What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of an HTML or XML document as a tree of objects, allowing scripts to dynamically access, manipulate, and modify its content, structure, and styles. Each element, attribute, and text is represented as a node in this tree. DOM enables interaction between JavaScript and the webpage's elements.

How do you select an element from the DOM?

To select elements from the DOM, you can use several JavaScript methods. The `document.getElementById("id")` method selects a single element based on its unique `id` attribute. Another option is `document.querySelector("selector")`, which selects the first element that matches a given CSS selector, while `document.querySelectorAll("selector")` selects all matching elements and returns a static NodeList. For selecting elements by class name, `document.getElementsByClassName("className")` returns a live HTMLCollection of all elements with the specified class. Similarly, `document.getElementsByTagName("tagName")` selects all elements of a given tag, such as `div` or `p`. Additionally, the `element.children` property retrieves all direct child elements of a specified parent element, allowing for easier navigation within the DOM hierarchy.

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where a single event listener on a parent element handles events from its child elements. This works through event bubbling, where events propagate up to the parent. It’s useful because it improves performance by reducing the number of event listeners and simplifies management, especially with dynamically added elements. Instead of attaching listeners to each child, the parent’s listener can manage all relevant child events efficiently.

How do you manipulate an element's attributes and styles using the DOM?

You can manipulate an element's attributes and styles using the DOM with JavaScript in several ways. To modify an attribute, you can use the `setAttribute()` method to change or add an attribute, while `getAttribute()` retrieves its value. For instance, `element.setAttribute("src", "image.jpg")` sets the `src` attribute of an image element, and `const value = element.getAttribute("src")` retrieves it. To remove an attribute, you can use the `removeAttribute()` method, such as `element.removeAttribute("class")`, which removes the class attribute entirely. Many attributes can also be accessed directly via their properties; for example, `element.id = "newID"` sets the `id` property. When it comes to styling, the `style` property allows you to modify inline CSS styles, such as `element.style.color = "red"` to change the text color. Additionally, you can use the `classList` property to manage CSS classes, allowing you to add classes with `element.classList.add("active")`, remove them with `element.classList.remove("hidden")`, or toggle them with `element.classList.toggle("highlight")`. This versatility enables dynamic and responsive changes to the webpage's elements.
